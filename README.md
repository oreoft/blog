## 博客配置与技术笔记

### 核心目录结构
- **_config.yml**: 网站的核心配置信息。
- **_posts**: 存放博客正文内容（Markdown文件）。
- **_pages**: 其他需要生成的独立网页，例如 About 页。
- **_layouts**: 网页排版模板（如 post, page, default）。
- **_includes**: 被模板包含的 HTML 片段（如 header, footer），可在 `_config.yml` 中修改位置。
  - `blog-page`: 分类布局
  - `jumbotron`: 封面页布局
- **_site**: Jekyll 最终编译生成的静态网页目录。
- **index.html**: 网站入口首页。

### 布局模版 (Layouts)
- **default**: 基础模版，只有文字，无 Banner。
- **page**: 有 Banner，无版权信息，无目录。
- **about**: 有 Banner，有目录，无版权信息。
- **post**: 标准文章页，有 Banner，有目录，有版权信息。

### 路由与编译规则
1. **文件命名**: 只有符合 `YYYY-MM-DD-title.md` 格式的文件才会被 Jekyll 编译。
2. **路径编译**: 新建文件夹并创建 md 后，只要格式符合，都会按照文件夹路径进行编译（除非在 md 头信息中手动重写 path）。
3. **多语言路径**:
   - `zh` 文件夹下的内容重写了路径，不包含 `zh` 前缀。
   - `en` 文件夹下的内容路径包含 `en/`（或者利用文件夹结构让其走默认路径），实现分别路由。
   - `_posts` 根目录下的内容默认无前缀。
4. **分类收录**: 每个分类的 HTML 页面（如 `java.html`）通过判断条件决定收录哪些文章，修改规则需编辑对应的 HTML 或在文章 `category` 中配合。
5. **Liquid 语法示例**:
   ```liquid
   {% if page.lang == 'en' %}
   {% elsif page.lang == 'zh' %}
   {% endif %}
   ```

## 写作风格与排版指南 (Based on 2019-2026)

### 1. 文章结构 (Structure)
我的文章通常遵循固定的三段式结构：
- **## 前言**: 
  - 必选。
  - 交代写作背景、契机（如“为了省钱”、“工作中遇到的坑”、“闲鱼淘了个垃圾”）。
  - 语气轻松，像在和朋友聊天。
- **## 正文**:
  - 分小标题 (`##`, `###`) 进行详细阐述。
  - 包含具体步骤、代码块、踩坑记录或硬件开箱图。
- **## 后言 / 总结 / 结束**:
  - 必选。
  - 总结体验、给出建议、或抒发一点个人感想（如“虽然折腾但乐在其中”）。
- **## 参考** (可选): 列出参考链接。

### 2. 语气与人设 (Tone)
- **第一人称**: 强烈个人色彩，使用“我”。
- **真实接地气**: 分享真实体验，不避讳“犯蠢”或“翻车”经历。
- **用词习惯**: 
  - 会使用一些互联网/极客俚语（如：垃圾佬、白嫖、u1s1、yysy、真香、挤牙膏）。
  - 态度谦虚但有主见（如：不迷信参数，看重实际体验）。

### 3. 排版规范 (Formatting)

#### Front Matter (文章头信息)
```yaml
---
layout: post
title: 文章标题
excerpt: 一句话摘要/副标题
category: [分类，如 other, java, linux]
keywords: [关键词，逗号分隔]
lang: zh
---
```

#### 图片 (Images)
- 使用 Markdown 插入图片。
- **必须**在图片下方紧跟 `<center>` 标签作为注释/图注。
- 通常在 `center` 后加 `<br>` 增加间距。
- 示例：
  ```markdown
  ![图片描述](图片链接)
  <center>这是图片的描述文字</center><br>
  ```

#### 引用与强调
- **引用 (`>`)**: 用于引用官方定义、详细参数列表或补充说明。
- **加粗 (`**text**`)**: 用于强调关键步骤、坑点或结论。
- **表格**: 喜欢使用 Markdown 表格对比参数或列出清单（如：硬件配置单、CPU后缀含义）。

#### 代码块
- 标明语言类型（如 `java`, `shell`, `xml`）。
- 如代码太长，会提示“代码已上传 GitHub”并给出链接。
